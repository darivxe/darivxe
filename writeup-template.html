Reconnaissance

The challenge presented a login page at /admin. Inspecting the source revealed a commented-out endpoint that returned user data without authentication.

// observation

The X-Debug-Token header was present in every response, leaking internal routing information.

Exploitation

Sending a crafted request to the debug endpoint with a manipulated Authorization header bypassed the authentication check entirely. 

$ curl -H "Authorization: debug" https://target.ctf/api/debug

import requests

Takeaways

This CTF reinforced the importance of always checking client-side validation assumptions. Several challenges shared a pattern of trusting headers that should never be trusted in production.

  • Always proxy traffic to catch hidden headers and endpoints.
  • Debug modes are left on more often than you'd expect in CTF environments.
  • A quick curl beats clicking around a UI every time.
// gotcha

The JWT challenge used alg: none — a classic but still effective trick. Never assume a CTF won't use well-known CVEs.
